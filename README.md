# Интерактивный обработчик задач

## Описание
Проект представляет собой платформу обработки задач, сфокусированную на правильной **инкапсуляции**, **валидации данных** и защите **инвариантов** доменной модели.

Основная цель работы — демонстрация возможностей Python по управлению доступом к атрибутам через:
*   **Дескрипторы (Descriptors)**: Как Data (`__set__` + `__get__`), так и Non-data (`__get__`).
*   **Свойства (@property)**: Для вычисляемых полей и защиты приватных атрибутов.

Проект построен по принципам **Clean Architecture** с четким разделением ответственности по слоям:
*   `domain`: Ядро системы. Содержит бизнес-сущности (`Task`), правила валидации (дескрипторы), исключения и интерфейсы. Не зависит от внешних библиотек.
*   `usecases`: Прикладная логика. Реализует конкретные сценарии использования (например, импорт задач), координируя работу домена и репозиториев.
*   `repository`: Реализация доступа к данным. Отвечает за получение и сохранение задач из конкретных источников (файлы, API, память).
*   `adapters`: Внешние интерфейсы. В данном случае — CLI, который взаимодействует с пользователем.

## Пользовательские дескрипторы
В проекте реализованы reusable-компоненты для валидации:
*   `PriorityValidator`: Data descriptor, проверяет, что приоритет — целое число от 1 до 5.
*   `DescriptionValidator`: Data descriptor, гарантирует, что описание — непустая строка.
*   `StatusInfoDescriptor`: **Non-data descriptor**, демонстрирующий механизм перекрытия атрибутов.
*   `FilePathValidator`, `TaskCountValidator`: Валидаторы для ввода данных в CLI.

## Основные используемые библиотеки

В проекте использовались следующие библиотеки:
- **questionary** — создание красивого интерактивного меню в командной строке
- **pytest & pytest-cov** — фреймворк для написания юнит-тестов и проверки процента покрытия кода
- **mypy** — статический анализатор для строгого контроля типов

## Поддерживаемые источники задач

Все источники автономны и реализуют общий контракт (метод `get_tasks() -> list[Task]`), описанный в доменном слое

| Источник | Описание |
|:---|:---|
| **TaskFile** | Загружает задачи из текстового файла. Ожидает формат: `ID JSON_PAYLOAD` в каждой строке |
| **TaskRandom** | Программно генерирует указанное количество задач со случайными атрибутами и приоритетами |
| **TaskAPI** | Упрощённая заглушка, имитирующая получение заранее заготовленных задач от внешнего веб-сервиса |

## Основные команды CLI

| Команда | Описание |
|:---|:---|
| **Задачи из файла** | Запрашивает путь к файлу, валидирует его (дескриптор `FilePathValidator`) и импортирует задачи |
| **Рандомные задачи** | Запрашивает желаемое количество задач, валидирует ввод (дескриптор `TaskCountValidator`) и генерирует их |
| **Задачи из API-заглушки** | Обращается к API-моку и моментально загружает заготовленные задачи в репозиторий |
| **Просмотреть список задач** | Выводит на экран все задачи, которые в данный момент хранятся в репозитории `TaskRepository` |
| **Выход из программы** | Безопасное завершение работы подсистемы |

## Установка и запуск

### Клонирование репозитория
    ```bash
    git clone https://github.com/moonshyXD/TaskHandler/tree/task-model
    cd TaskHandler
    ```
### Установка зависимостей
    ```bash
    make build
    ```
### Запуск приложения
    ```bash
    make run
    ```

## Тестирование и проверка качества

### Запуск тестов
    ```bash
    pytest
    ```
### Проверка покрытия тестами
    ```bash
    make testcover
    ```
