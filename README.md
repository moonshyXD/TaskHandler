# Интерактивный обработчик задач с проверкой контрактов

## Описание
Интерактивная подсистема приёма и обработки задач из различных источников, демонстрирующая работу Duck Typing и контрактного программирования в Python.

В системе реализован сбор задач из нескольких независимых источников, которые не связаны общим базовым классом/наследованием, но обязаны реализовывать единый поведенческий контракт `Protocol`. 

Проект построен по принципам **Clean Architecture** с разделением на слои: `domain`, `usecases`, `adapters`.
Реализована строгая рантайм-проверка контрактов (`@runtime_checkable`), продвинутая типизация и кастомные дескрипторы для валидации вводимых данных.

## Зависимости

Для установки зависимостей для разработчика выполните команду:
```bash
make build
```

## Основные используемые библиотеки

В проекте использовались следующие библиотеки:
- **questionary** — создание красивого интерактивного меню в командной строке
- **pytest & pytest-cov** — фреймворк для написания юнит-тестов и проверки процента покрытия кода
- **mypy** — статический анализатор для строгого контроля типов

## Поддерживаемые источники задач

Все источники автономны и реализуют общий контракт (метод `get_tasks() -> list[Task]`), описанный в доменном слое

| Источник | Описание |
|:---|:---|
| **TaskFile** | Загружает задачи из текстового файла. Ожидает формат: `ID JSON_PAYLOAD` в каждой строке |
| **TaskRandom** | Программно генерирует указанное количество задач со случайными атрибутами и приоритетами |
| **TaskAPI** | Упрощённая заглушка, имитирующая получение заранее заготовленных задач от внешнего веб-сервиса |

## Основные команды CLI

| Команда | Описание |
|:---|:---|
| **Задачи из файла** | Запрашивает путь к файлу, валидирует его (дескриптор `FilePathValidator`) и импортирует задачи |
| **Рандомные задачи** | Запрашивает желаемое количество задач, валидирует ввод (дескриптор `TaskCountValidator`) и генерирует их |
| **Задачи из API-заглушки** | Обращается к API-моку и моментально загружает заготовленные задачи в репозиторий |
| **Просмотреть список задач** | Выводит на экран все задачи, которые в данный момент хранятся в репозитории `TaskRepository` |
| **Выход из программы** | Безопасное завершение работы подсистемы |

## Руководство по запуску

Склонируйте Github репозиторий командой:
```bash
git clone https://github.com/moonshyXD/TaskHandler
```

Перейдите в папку с репозиторием:
```bash
cd TaskHandler
```

Установите [зависимости](#зависимости)

Активируйте виртуальное окружение:
```bash
source .venv/bin/activate
```

Для успешного запуска программы следует прописать в командной строке команду:
```bash
make run
```
ИЛИ
```bash
uv run main.py
```

## Тестирование
Запустите виртуальное окружение, если оно до этого не было включено командой:
```bash
source .venv/bin/activate
```

Для запуска тестов программы пропишите в командной строке команду:
```bash
pytest
```

Для просмотра того, сколько % кода покрыто тестами, пропишите в командной строке:
```bash
make testcover
```
